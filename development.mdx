<Info>
  **Prerequisites**:
  - Node.js version 19 or higher
  - A docs repository with a `docs.json` file
</Info>

Use this page when you are editing the docs repo itself.

<Steps>
<Step title="Install the Mintlify CLI">

```bash
npm i -g mint
```
</Step>

<Step title="Preview locally">
Navigate to the docs repo and run:

```bash
mint dev
```

A local preview opens on `http://localhost:3000` by default.

</Step>
</Steps>

## Custom ports

If port `3000` is already in use by the main app, run Mintlify on a different port:

```bash
mint dev --port 3333
```

## Validating links

Run a broken-link pass before you commit:

```bash
mint broken-links
```

## Recommended workflow

1. Run `mint dev --port 3333` if the app is already on `3000`
2. Check the main docs routes you changed
3. Run `mint broken-links`
4. Commit and push to the docs repo default branch or your review branch

## Shipping to app.outfox.ai/docs

The docs content lives in the Mintlify repo, but the public path target is:

```text
https://app.outfox.ai/docs
```

For the subpath setup, use the Vercel rewrite flow documented in
[/platform/deployment-vercel](/platform/deployment-vercel).
